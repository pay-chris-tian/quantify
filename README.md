# Quantify

## 개요

Quantify는 **국내 주식 시장을 대상으로 한 개인용 퀀트 투자 연구 및 시스템 개발 프로젝트**입니다.

이 프로젝트의 1차 목표는 자동매매를 빠르게 만드는 것이 아니라,  
**투자 아이디어를 명시적인 규칙으로 정의하고, 데이터를 통해 검증 가능한 형태로 바꾸는 것**입니다.

즉, Quantify는 다음을 목표로 합니다.

1. 투자 아이디어를 숫자와 규칙으로 표현한다.
2. 과거 데이터로 전략을 백테스트한다.
3. 성과와 리스크를 투명하게 평가한다.
4. 충분한 검증 이후에만 페이퍼 트레이딩과 자동화를 검토한다.

---

## 프로젝트 철학

Quantify는 아래 원칙을 따릅니다.

### 1. 자동화보다 전략 검증이 먼저
검증되지 않은 전략을 자동화하면 실수를 더 빠르게 반복하게 됩니다.

### 2. 직관보다 정량화
“좋아 보인다”, “흐름이 좋아 보인다” 같은 표현은  
검증과 자동화가 어렵습니다.  
Quantify는 이를 가능한 한 데이터와 규칙으로 번역합니다.

### 3. 복잡함보다 설명 가능성
초기 단계에서는 복잡한 예측 모델보다  
설명 가능하고 반복 가능한 팩터 기반 전략을 우선합니다.

### 4. 수익률만이 아니라 리스크도 본다
전략 평가는 누적 수익률뿐 아니라  
최대낙폭(MDD), 변동성, 샤프 비율, turnover 등을 함께 봅니다.

### 5. 감정 개입 최소화
투자 판단을 최대한 규칙화하여  
감정적 개입과 즉흥적 판단을 줄이는 것을 지향합니다.

---

## 현재 전략 방향

Quantify의 현재 전략 방향은 **멀티팩터 퀀트 전략**입니다.

초기 전략은 아래 세 축을 결합하는 방향으로 설계합니다.

- **Value**: 상대적으로 저평가된 종목 선별
- **Momentum**: 중기적으로 강한 추세를 보이는 종목 선호
- **Quality**: 재무적으로 비교적 우수한 기업 선호

즉, 기존의 단일 가치 전략에서 출발하되,  
현재는 **가치 + 모멘텀 + 퀄리티를 결합한 검증 가능한 포트폴리오 전략**으로 발전시키는 것이 핵심입니다.

---

## 투자 대상

초기 범위는 아래와 같습니다.

- 국내 주식 시장 (KOSPI / KOSDAQ)
- 보통주 중심
- 거래정지/관리종목/유동성 부족 종목은 제외 가능
- 추후 ETF 또는 별도 유니버스 확장 검토

---

## 전략 개요

### 팩터 후보

#### Value
- PER
- PBR
- PSR
- EV/EBITDA (데이터 정합성 확보 시)

#### Momentum
- 3개월 수익률
- 6개월 수익률
- 12개월 수익률

#### Quality
- ROE
- 영업이익률
- 부채비율
- 이익 안정성 관련 지표

#### Liquidity / Filter
- 평균 거래대금
- 최소 시가총액
- 적자 기업 제외 여부
- 금융주 제외 여부

### 기본 포트폴리오 가설

초기 가설은 다음과 같습니다.

1. 유동성이 낮은 종목을 제외한다.
2. 가치, 모멘텀, 퀄리티 팩터를 점수화한다.
3. 표준화 혹은 순위 기반으로 합산 점수를 만든다.
4. 상위 N개 종목을 동일 비중으로 보유한다.
5. 월 1회 리밸런싱한다.

이 구조는 최종 확정안이 아니라 **백테스트로 검증할 출발점**입니다.

---

## 리스크 관리 원칙

Quantify는 초기부터 개별 종목 손절 규칙보다  
**포트폴리오 차원의 리스크 관리**를 우선합니다.

예시:
- 종목 수 분산
- 유동성 필터
- 팩터 분산
- 리밸런싱 규칙 유지
- 거래비용 반영
- 최대낙폭(MDD) 기준 평가

즉, 단기 트레이딩식 익절/손절보다  
**규칙 기반 교체와 포트폴리오 관리**를 중심으로 설계합니다.

## 개발 우선순위

### Phase 1. 전략 명세
- 투자 가능 유니버스 정의
- 팩터 정의
- 리밸런싱 주기 정의
- 성과 지표 정의
- 거래비용 가정 정의

### Phase 2. 데이터 레이어
- 가격 데이터 수집/정제
- 재무제표 데이터 정리
- 종목 메타데이터 정리
- 데이터 누수 방지 구조 검토

### Phase 3. 백테스트 MVP
- 유니버스 생성
- 팩터 계산
- 랭킹 및 포트폴리오 구성
- 리밸런싱
- 거래비용 반영
- 성과 분석

### Phase 4. 검증
- 벤치마크 비교
- 민감도 테스트
- 기간 분할 검증
- 팩터 기여도 점검

### Phase 5. 리포트 / 대시보드
- 월별 편입/편출 종목
- 점수 요약
- 자산곡선
- 성과 지표 요약
- 전략 변경 로그

### Phase 6. 페이퍼 트레이딩
- 실거래 없는 시그널 운영
- 체결 가능성 확인
- 슬리피지 및 운영 이슈 확인

### Phase 7. 자동화
- API 연동
- 주문 로직
- 체결 확인
- 실패 처리
- 알림 및 로그

## 현재 기술 방향

초기 기술 스택은 다음 정도를 가정합니다.

- 데이터 처리: `pandas`, `numpy`
- 시각화/리포트: `matplotlib`, `plotly`, `streamlit`
- 데이터 소스 연동: `OpenDART`, 웹 수집 도구 등
- 백테스트: 초기엔 커스텀 로직 우선, 필요 시 `backtrader` 검토
- 자동화: 추후 `FastAPI`, 스케줄러, 알림 시스템 검토

초기 단계에서는 프레임워크 도입보다  
**데이터 정합성과 전략 검증 로직**이 더 중요합니다.

## 성공 기준

현재 단계에서 성공은 “자동매매 완성”이 아닙니다.

현재 단계의 성공은 다음입니다.

- 전략이 명시적으로 정의되어 있다.
- 규칙이 코드로 표현 가능하다.
- 백테스트가 가능하다.
- 결과를 설명할 수 있다.
- 개선 이력이 남는다.

## 주의사항

이 프로젝트는 개인 연구 및 학습 목적의 자산 운용 시스템입니다.  
실제 투자 성과를 보장하지 않으며, 투자 판단의 최종 책임은 사용자에게 있습니다.
